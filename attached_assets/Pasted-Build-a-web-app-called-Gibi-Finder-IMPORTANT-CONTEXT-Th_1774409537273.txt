Build a web app called Gibi Finder.

IMPORTANT CONTEXT

This project will be created and run inside Replit.
Deployment will also be done on Replit.

Do not build a complex monorepo.

- Python backend with FastAPI

=====================================
MAIN GOAL
=====================================

Create a comic-book identification web app.

The app must allow the user to:
- upload comic images
- upload up to 3 images in one search
- remove uploaded images before searching
- search by text description
- search by character
- search by quote / balloon text
- see a main result
- see related results
- rate if the answer is correct
- suggest a correction
- view history
- view ranking
- open a public result page

The app should feel like a “Shazam for comic books”.

IMAGE SEARCH
=====================================

Allow uploading:
- 1 image
- 2 images
- 3 images maximum

UI requirements:
- drag and drop area
- click to upload
- image preview
- each preview has its own X button
- X removes the specific image instantly
- user can add another image after removing one
- if user adds more than 3, show friendly error

Backend:
- send all images together to Gemini in a single request

API route:
POST /api/identify

Response structure:
{
  "mainResult": {
    "id": "...",
    "encontrado": true,
    "revista": "",
    "titulo": "",
    "editora": "",
    "ano": "",
    "pagina": "",
    "personagens": [],
    "descricao": "",
    "confianca": 0,
    "nota": "",
    "balloon_text": ""
  },
  "relatedResults": []
}
